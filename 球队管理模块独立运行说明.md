# 模块独立运行说明

## 项目结构概述

本项目采用前后端分离架构，分为两个独立模块：

1. **后端模块**：位于 `backend` 目录，使用 Spring Boot 框架开发
2. **前端模块**：位于 `frontend` 目录，使用 Vue.js 框架开发

两个模块可以完全独立编译、构建和运行，互不依赖。

## 后端模块独立运行

### 环境要求

- JDK 1.8 或以上版本
- Maven 3.0 或以上版本
- MySQL 5.7 或以上版本（需要预先创建数据库）

### 配置步骤

1. **数据库配置**：
   - 修改 `backend/src/main/resources/application.yml` 文件中的数据库连接信息：
     ```yaml
     spring:
       datasource:
         url: jdbc:mysql://localhost:3306/football?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
         username: root
         password: 123456
         driver-class-name: com.mysql.cj.jdbc.Driver
     ```

### 编译与运行

1. **编译项目**：
   ```bash
   cd backend
   mvn compile
   ```

2. **打包项目**：
   ```bash
   mvn package
   ```

3. **运行项目**：
   - 方式一：使用 Maven 直接运行
     ```bash
     mvn spring-boot:run
     ```
   - 方式二：使用打包后的 JAR 文件运行
     ```bash
     java -jar target/football-league-system-1.0.0.jar
     ```

4. **验证运行状态**：
   - 访问 `http://localhost:8080`，如果看到后端 API 响应，则说明后端服务已成功启动

## 前端模块独立运行

### 环境要求

- Node.js 16.0 或以上版本
- npm 8.0 或以上版本

### 配置步骤

1. **API 代理配置**：
   - 修改 `frontend/vue.config.js` 文件中的代理配置（指向后端服务地址）：
     ```javascript
     module.exports = {
       devServer: {
         port: 3000,
         proxy: {
           '/api': {
             target: 'http://localhost:8080',
             changeOrigin: true,
             pathRewrite: {
               '^/api': ''
             }
           }
         }
       }
     }
     ```

### 编译与运行

1. **安装依赖**：
   ```bash
   cd frontend
   npm install
   ```

2. **开发模式运行**：
   ```bash
   npm run serve
   ```

3. **生产环境构建**：
   ```bash
   npm run build
   ```
   - 构建产物将生成在 `frontend/dist` 目录下

4. **验证运行状态**：
   - 访问 `http://localhost:3000`，如果看到前端页面，则说明前端服务已成功启动

## 模块独立测试

### 后端 API 测试

可以使用 Postman 或 curl 工具测试后端 API：

```bash
# 示例：获取球队列表
curl http://localhost:8080/team/query_page?pageNum=1&pageSize=10
```

### 前端界面测试

直接访问 `http://localhost:3000`，可以测试前端界面功能，包括：
- 登录/注册
- 球队管理
- 球员管理
- 比赛管理等

## 注意事项

1. **端口冲突**：
   - 默认情况下，后端服务使用 8080 端口，前端服务使用 3000 端口
   - 如果端口被占用，可以在各自的配置文件中修改端口号

2. **数据库初始化**：
   - 首次运行后端服务时，需要确保 MySQL 数据库已创建
   - 可以使用项目中的 SQL 脚本初始化数据库结构和测试数据

3. **依赖安装**：
   - 前端模块首次运行前需要执行 `npm install` 安装依赖
   - 后端模块首次运行前会自动下载 Maven 依赖

4. **跨域问题**：
   - 开发模式下，前端通过代理配置解决跨域问题
   - 生产环境部署时，需要确保后端服务已配置 CORS 或使用相同的域名

## 总结

本项目的前后端模块已实现完全独立，可以根据需要分别编译、构建和运行。独立运行模式适用于：
- 开发阶段的模块化开发和测试
- 单独部署某个模块
- 针对某个模块进行性能优化和调试

通过上述步骤，您可以轻松实现前后端模块的独立运行和测试。
